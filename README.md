# Prevent Bot Abuse

A lightweight machine learning system for detecting abusive or adversarial user prompts in banking chatbots.

The project uses sentence embeddings generated by **all-MiniLM-L6-v2** and a **Logistic Regression** classifier to distinguish between legitimate customer queries and misuse attempts.

## Motivation

Banking chatbots are vulnerable to prompt abuse, toxic inputs, and adversarial attempts. Such interactions can:

- Produce undesirable outputs
- Disrupt customer experience
- Increase operational risks
- Lead to prompt injection or jailbreak attempts

This project aims to build a fast and interpretable classifier capable of filtering malicious or irrelevant inputs before they reach downstream systems.

---

## Features

- Combines multiple datasets of genuine and abusive prompts.
- Generates semantic embeddings using Sentence Transformers.
- Trains a Logistic Regression classifier.
- Achieves approximately **94% test accuracy**.
- Identifies false positives and false negatives for error analysis.
- Visualizes embedding space using UMAP.

## Dataset

The classifier is trained using:

### Genuine Queries
- Banking chatbot customer interactions (`cleaned_data.csv`)

### Abuse Queries
- Toxic chat dataset
- Banking chatbot adversarial prompts

Labels:

| Label | Meaning |
|---------|--------|
| 1 | Genuine customer query |
| 0 | Misuse / toxic / adversarial query |

---

## Generate Embeddings

```bash
python create_embeddings.py
```

Uses:

```
SentenceTransformer("all-MiniLM-L6-v2")
```

---

## Train Model

```bash
python train_model.py
```

Model:

- Logistic Regression
- max_iter = 2000

## Performance

### Test Accuracy

```
94.44%
```

Classification report:

| Class | Precision | Recall | F1 |
|---------|-----------|--------|----|
| Misuse (0) | 0.96 | 0.88 | 0.92 |
| Genuine (1) | 0.94 | 0.98 | 0.96 |
